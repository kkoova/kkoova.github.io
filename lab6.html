Лабораторная работа №6

Тестирование веб-приложения на проникновение методом полного перебора

1. Цель работы

Получить практические навыки тестирования веб-приложения на устойчивость к атакам методом полного перебора (brute-force). Убедиться в ненадежности простых паролей, в необходимости применения дополнительных технологий проактивной защиты учётных данных.

  • Выводы о проделанной работе (вкл. рекомендации по предотвращению проникновения методом полного перебора)

2. Теоретические сведения

2.1 Brute-force

Brute-force (атака полным перебором) – метод решения математических задач, сложность которого зависит от количества всех возможных решений. Сам же термин brute-force обычно используется в контексте хакерских атак, когда злоумышленник пытается подобрать логин/пароль к какой-либо учетной записи или сервису.

2.2 Тест на проникновение

Тест на проникновение — это практика попыток взломать собственные ИТсистемы, используя те же способы, что и хакеры. Это позволяет выявлять любые уязвимости.

2.3 Цель тестирования

В данной работе в качестве тестируемого веб-приложения используется DVWA. Damn Vulnerable Web Application (DVWA) — это веб-приложение на PHP/MySQL, которое содержит множество уязвимостей. Его главная цель — помочь профессионалам по безопасности протестировать их навыки и инструменты в легальном окружении, помочь веб-разработчикам лучше понять процесс безопасности веб-приложений и помочь и студентам и учителям в изучении безопасности веб-приложений в контролируем окружении аудитории.

3. Установка инструментов

! ВНИМАНИЕ: DVWA не эмулирует уязвимости веб-приложений, уязвимости внутри DVWA являются настоящими, поэтому работайте с ним только на виртуальной машине. Не загружайте его на любой сервер, имеющий выход в Интернет.

Шаг 0. hydra

В данной работе для атаки перебором используется программа, предустановленная в Kali Linux – hydra.
Справку по имеющимся опциям можно получить, выполнив команду (она понадобиться вам в будущем):

hydra –h

Шаг 1. Запустите приложение DVWA

sudo docker run docker.io/vulnerables/web-dvwa:latest

Далее необходимо перейти по URL-адресу, указанному в подробном выводе терминала

Учётные данные для входа в DVWA:

  • Логин: admin
  • Пароль: password

Нажмите кнопку «Create / Reset Database». Перезапустите приложение.

Шаг 2. Откройте Burp Suite. Его можно найти в списке приложений Kali Linux > Web Applications > Web Application Proxies > burpsuite

Запустите временный проект. Откройте Proxy > Intercept, как на скриншоте ниже. Откройте настроенный для работы браузер (кнопка «Open browser»). В открытый браузер скопируйте URL открытого DVWA. Введите логин и пароль.



Выставите уровень безопасности «Low» (На вкладке «DVWA Security») Затем перейдите во вкладку «Brute Force».

Все готово! Можно начинать перехватывать трафик.




Перейдите в BurpSuite (не закрывайте браузер, сверните его). Включите перехват «Intercept is on».




Перейдите в браузер в DVWA и введите логин «admin» и произвольный пароль. Нажмите кнопку «Login».




Вас перебросит в BurpSuite на вкладку перехватчика. Нажмите на кнопку «Forward», и перейдите на вкладку «HTTP History». Не закрывая BurpSuite, разверните браузер и проверьте, что на веб-форме появилось сообщение об ошибке.




Таким образом вы получили всю необходимую информацию для составления команды перебора паролей. Выключите перехват.
