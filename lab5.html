Активный анализ сетевой инфраструктуры

1. Цель работы

Ознакомиться с некоторыми инструментами для анализа безопасности вебприложений и сканирования их на уязвимости. Приобрести практические навыки проведения анализа. Проведите анализ веб-приложений DVWA и OWASP Juice Shop при помощи следующих инструментов (Whatweb, Nikto, OWASP Zed Attack Proxy)

Содержание отчета

  • Титульный лист
  • Задание
  • Результаты анализа приложения с помощью whatweb
  • Результаты анализа приложения с помощью nikto
  • Результаты анализа приложения с помощью OWASP Zed Attack Proxy (ZAP)
  • Вывод
Внимание! За несвоевременное представление работы на проверку оценка снижается на 1 балл!

2. Теоретические сведения

2.1 Активный анализ

Активный анализ — это метод исследования сети, при котором исследователь отправляет специальные сетевые пакеты к узлам сети для получения информации об их состоянии.

В отличие от пассивного анализа, активный анализ взаимодействует с сетью, создавая сетевой трафик.

2.2 Основные задачи активного анализа

  • обнаружение активных устройств
  • сканирование портов
  • определение сетевых сервисов
  • поиск уязвимостей

3. Установка инструментов

Шаг 0. Настройка

В настройках виртуальной машины
Устройства -> Сеть -> Настроить сеть -> Выбираем сетевой мост

sudo apt update

Шаг 1. Установите приложение DVWA

sudo apt install -y docker.io

sudo systemctl enable docker --now

sudo docker search web-dvwa 

sudo docker pull vulnerables/web-dvwa 

sudo docker run docker.io/vulnerables/web-dvwa:latest

Далее необходимо перейти по URL-адресу, указанному в подробном выводе терминала

Учётные данные для входа в DVWA:

  • Логин: admin
  • Пароль: password

Нажмите кнопку «Create / Reset Database». Перезапустите приложение.

Шаг 2. Установите и запустите приложение OWASP Juice Shop

sudo apt install -y docker.io
    
sudo systemctl enable docker --now

sudo docker pull bkimminich/juice-shop

sudo docker run -d -p 3000:3000 --name juice-shop bkimminich/juice-shop

После запуска контейнер доступен в веб-браузере: в Linux — по адресу http://localhost:3000

Шаг 3. Whatweb

whatweb цель(DVWA\OWASP Juice Shop) –v –a 1

Запишите в отчет сведения, которые удалось получить: какие используются веб-технологии, веб-сервер, библиотеки, перенаправления, интересные для тестирования поля и т.п.

Шаг 4. Nikto

 nikto –h цель(DVWA\OWASP Juice Shop) –p порт(DVWA\OWASP Juice Shop)

Проанализируйте результат сканирования. Запишите в отчет результаты анализа.

Шаг 5. OWASP Zed Attack Proxy (ZAP)

sudo apt install zaproxy

Запустите автоматическое сканирование (установите «use traditional spider»)


Изучите результаты сканирования. Во вкладке «sites» разверните папку с исследуемым сайтом, чтобы просмотреть файлы, которые обнаружил инструмент «Spider».

Откройте вкладку «Alerts». Просмотрите найденные уязвимости. Уязвимости уровней «High» и «Medium» изучите подробно, при возможности попробуйте поэксплуатировать.

Результаты каждого действия должны быть зафиксированы скриншотом и добавлены в отчет.
